## 2. 背景と目的

### 背景

- 夢や目標を持っていても、日常に埋もれて行動に移せない人が多い
- 夢を言語化し、具体的なステップに分解し、日々の行動として実行する仕組みが必要
- 既存のタスク管理ツールは「効率化」に焦点を当てており、「夢の実現」という視点が欠けている

### 目的

- ユーザーが夢を言語化し、目標→タスクに分解して実行できるアプリを提供する
- 活動の継続をゲーミフィケーション（星座システム）で支援する
- Web体験版で気軽に始められる環境を提供する

### 戦略的位置づけ

- 収益化プロセスを学ぶ場（布教→利用→サブスク契約の導線構築）
- AI駆動開発のポートフォリオ
- ユーザーと一緒に育てるプロダクト

## 4. 機能要件

### 4.1 コア機能

| ID | 機能 | 説明 | 優先度 |
|---|---|---|---|
| F-001 | 夢の管理 | 登録・編集・削除・ドラッグ並び替え。カテゴリ分類（8種） | 必須 |
| F-002 | 目標の管理 | 夢に紐づく目標の登録・編集・削除・ドラッグ並び替え。What/When/How | 必須 |
| F-003 | タスクの管理 | 目標に紐づくタスクの登録・編集・削除。開始日/終了日/進捗率 | 必須 |
| F-004 | 活動予定表示 | タスクをタイムライン形式で可視化。今日の帯表示・日付ジャンプ | 必須 |
| F-005 | 活動ログ記録 | タスクごとの活動時間を記録 | 必須 |
| F-006 | 書籍管理 | 登録・ステータス管理・ソート（登録日/更新日/50音）・読了レビュー | 必須 |
| F-007 | 統計表示 | 活動時間・日数・連続記録・目標別/書籍別統計 | 必須 |
| F-008 | ダッシュボード | カスタマイズ可能なウィジェットグリッド（12種パーツ） | 必須 |

## 5. 非機能要件

### 5.1 パフォーマンス

| ID | 要件 | 基準 |
|---|---|---|
| NF-001 | 全操作の応答速度 | 1秒以内（ユーザー体感ベース） |
| NF-002 | 活動予定スクロール | 指追従でシームレス（フレーム落ちなし） |
| NF-003 | DB問い合わせ | N+1クエリ禁止。一括取得方式 |
| NF-004 | CustomPainter | Paint オブジェクトをコンストラクタでキャッシュ |
| NF-005 | ストレステスト | 週次自動実行（36テスト、閾値超過時にIssue自動起票） |

### 5.2 セキュリティ

| ID | 要件 | 基準 |
|---|---|---|
| NF-006 | XSS対策 | ユーザー入力をHTML/CSSに埋め込む箇所はサニタイズ |
| NF-007 | SQLインジェクション対策 | Drift ORMパラメータ化クエリのみ使用 |
| NF-008 | 入力バリデーション | 全入力フィールドにバリデーション設定 |
| NF-009 | インポート検証 | JSONインポート時にサイズ上限（10MB）・型チェック |
| NF-010 | サブスク検証 | URLパラメータだけでは有効化不可。サーバー検証必須 |
| NF-011 | 空データ上書き防止 | ローカルDBが空の場合、クラウドバックアップを上書きしない |

### 5.3 保守性

| ID | 要件 | 基準 |
|---|---|---|
| NF-012 | ラベル外出し | 全UIテキストを `app_labels.dart` で一元管理（約700定数） |
| NF-013 | テストカバレッジ | 全ソースコードにテスト対応（826テスト） |
| NF-014 | 静的解析 | `flutter analyze` エラー0件 |
| NF-015 | CI/CD | GitHub Actions で自動テスト・デプロイ |
| NF-016 | DBマイグレーション | カラム追加時は5点セット（テーブル・バージョン・マイグレーション・モデル・エクスポート） |

## 6. 制約条件

- Web版はGitHub Pages（静的ホスティング）で提供するため、サーバーサイド処理は不可
- データベースはクライアントサイドSQLite（Drift ORM + WASM）
- Firebase は認証とFirestore同期のみ使用（サーバーキー禁止）
- Stripe 決済は Google Apps Script 経由で処理
- 全サービスを無料枠内で運用（コスト発生閾値: DAU 3,000人超）

## 7. 用語定義

| 用語 | 定義 |
|---|---|
| 夢 | ユーザーの長期的な願望・ビジョン |
| 目標 | 夢を実現するための具体的なステップ（What/When/How） |
| タスク | 目標を達成するための日々の行動項目 |
| 活動予定 | タスクのタイムライン表示 |
| 活動ログ | タスクに対する活動時間の記録 |
| 星座 | 活動の積み重ねで完成するゲーミフィケーション要素 |
| 受信ボックス | リマインダー・実績・開発者通知の統合受信箱 |
| 発見ガイド | 質問回答形式で夢/目標/タスクを提案する機能（リニューアル中） |
