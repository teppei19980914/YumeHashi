# ユメログ 仕様書

## 1. テーマとコンセプト

### テーマ

**「夢を持つことで、大きな壁も乗り越えられる」**

### コンセプト

ユメログは **「止まっている人を動かし、動いている人を支える」** アプリです。

人は慣性で行動します。動いている人は動き続け、止まっている人は止まり続けます。ユメログは「最初の一歩」を踏み出すきっかけと、動き続けるための仕組みを提供します。

#### 開発の原点

開発者は学生時代に夢を持てず、将来に希望を見出せない経験をしました。本との出会いを通じて「生き生きと生きるには夢が必要」と気づき、夢を言語化し行動に移す仕組みをアプリとして形にしました。

#### 3つの柱

1. **言語化**: 夢を言葉にして書き出すことで、脳がそれを「目標」として認識する
2. **分解**: 夢を目標→タスクに分解し、日々の行動レベルまで具体化する
3. **継続**: 活動の積み重ねを星座として可視化し、モチベーションを維持する

#### 設計思想

- **高機能をシンプルに**: 誰でも直感的に使えるUI。全画面統一デザイン
- **ハードコーディング排除**: 全テキストを外部ファイルで管理し、保守性を最大化
- **ユーザーと育てる**: フィードバック機能で意見を募り、継続的に改善する

## 2. 画面仕様

### 2.1 共通UI仕様

| 項目 | 仕様 |
|---|---|
| ヘッダー | 説明テキスト（左）+ `⋮` メニュー（右）+ `+` FAB（右端） |
| 追加ボタン | `FloatingActionButton.small`（全画面統一） |
| 補助機能 | `PopupMenuButton`（`⋮` メニュー）に統合 |
| 発見ガイド | 全画面でグレーアウト + 「開発中」表示 |

### 2.2 ダッシュボード（ホーム）

| 項目 | 仕様 |
|---|---|
| パス | `/` |
| 機能 | カスタマイズ可能なウィジェットグリッド |
| ウィジェット | 12種類（受信ボックス、今日の活動、合計時間、活動日数、夢の数、目標数、連続活動、本棚、自己ベスト、実施率、アクティビティ、星座） |
| 編集モード | 「カスタマイズ」ボタン（ラベル付き）→ ドラッグ並べ替え・サイズ変更・追加/削除 |
| 時間帯挨拶 | 0-5時: お疲れさまです、5-12時: おはようございます、12-18時: こんにちは、18時以降: こんばんは |

### 2.3 夢ページ

| 項目 | 仕様 |
|---|---|
| パス | `/dreams` |
| 登録項目 | タイトル（必須）、説明、Why（動機）、カテゴリ（8種） |

### 2.4 目標ページ

## 3. 受信ボックス仕様

### 通知種別

| 種別 | アイコン | データソース | 動作 |
|---|---|---|---|
| リマインド | アラーム（赤） | ユーザーのタスク/目標期限 | アプリ起動時に自動チェック |
| 実績 | トロフィー（黄） | ユーザーの活動記録 | マイルストーン達成時に自動生成 |
| お知らせ | メガホン（青） | `announcements.json` | 差分同期（既読状態保持） |

### 初回アクセスユーザーへの配慮

初回アクセス時は最新の1件のみ未読表示。過去の通知は既読としてDB登録。

## 4. 体験版制限仕様

### レベル別制限値

| レベル | 条件 | 夢 | 目標（全体） | タスク（各目標） | 書籍 |
|---|---|---|---|---|---|
| 0 | 初期状態 | 1 | 3 | 3 | 3 |
| 1 | フィードバック1回 | 2 | 5 | 5 | 4 |
| 2 | フィードバック2回 | 3 | 8 | 8 | 5 |
| 3+ | フィードバック3回以上 | 無制限 | 無制限 | 無制限 | 無制限 |

### プレミアム判定（5つの経路）

| 経路 | 判定方法 | 対象 |
|---|---|---|
| 非Web | `!isTrialMode` | ローカル実行時 |
| 招待コード | `_invitePremium` | 招待ユーザー |
| Stripe契約 | `_subscriptionPremium`（**サーバー検証必須**） | 契約者 |
| 無料トライアル | `_trialPremium`（7日間） | 体験ユーザー |
| 開発者メール | `_developerMode` | 開発者 |

## 5. チュートリアル仕様

### ステップ

| # | ステップ | 操作 |
|---|---|---|
| 1 | 夢ページへ移動 | ドロワーナビゲーション |
| 2 | 夢を追加 | ガイド選択（開発中）or 自分で入力 |
| 3 | 目標ページへ移動 | ドロワーナビゲーション |
| 4 | 目標を追加 | ガイド選択（開発中）or 自分で入力 |
| 5 | 活動予定ページへ移動 | ドロワーナビゲーション |
| 6 | タスクを追加 | ガイド選択（開発中）or 自分で入力 |
| 7 | AppBar説明 | 各アイコンの説明 |
| 8 | 完了 | データ保持 or 削除を選択 |

### データ追跡

チュートリアル中に作成したデータ（夢ID・目標ID・タスクID）を記録し、削除選択時にタスク→目標→夢の順で確実に削除する。

## 6. データ同期仕様

### クラウド同期（Firestore）

| 項目 | 仕様 |
|---|---|
| データ構造 | `users/{uid}` ドキュメントに全データJSON格納 |
| 同期方式 | 3秒デバウンス + SHA-256ハッシュ比較 |
| 起動時 | タイムスタンプ比較で自動ダウンロードまたはアップロード |
| 安全策 | 空データでクラウドを上書きしない |

### エクスポート/インポート

| 項目 | 仕様 |
|---|---|
| 形式 | JSON（全エンティティ含む） |
| 保持項目 | 夢・目標の sortOrder、書籍の全フィールド |
| サイズ上限 | 10MB |
