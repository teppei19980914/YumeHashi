# ユメハシ インフラ構成 & コスト検証

## 概要

ユメハシはサーバーレス・クライアント完結型の Web アプリケーションです。
ホスティングからデータ同期まで、すべて無料枠のサービスで構成されています。

---

## インフラ構成図

```
[ユーザー]
    |
    | HTTPS
    v
[GitHub Pages] ─── 静的ホスティング（Flutter Web ビルド成果物）
    |
    |── [Firebase Auth] ─── 匿名認証 / メール認証
    |
    |── [Firestore] ─── クラウドデータ同期（JSON全量同期）
    |
    |── [Google Apps Script] ─── Stripe プロキシ / フィードバック受付
    |       |
    |       └── [Stripe] ─── サブスクリプション決済
    |
    |── [GitHub Gist API] ─── リモート設定（招待コード / ユーザー設定）
    |
    └── [ブラウザ内 SQLite (WASM)] ─── ローカルデータベース（Drift ORM）
```

### 3. Cloud Firestore（クラウドデータ同期）

| 項目 | 内容 |
|------|------|
| 用途 | ユーザーデータのクラウドバックアップ・端末間同期 |
| データ構造 | `users/{uid}` ドキュメントに全データ JSON を格納 |
| 同期方式 | 3秒デバウンス + ハッシュ比較による差分検出 |
| 無料枠 | 読み込み 50,000回/日、書き込み 20,000回/日、ストレージ 1GB |

**同期トリガー:**
- アプリ起動時: タイムスタンプ比較で読み込みまたは書き込み（1回）
- データ変更時: 3秒デバウンス後、ハッシュ変更時のみ書き込み（1回）
- アプリ離脱時: 未同期データがあれば即座に書き込み（1回）

**安全策（実装済み）:**
- 3秒デバウンス（連続操作を1回にまとめる）
- SHA-256 ハッシュ比較（データ未変更なら書き込みスキップ）
- 空データによるクラウド上書き防止

### 現在のコスト: 0円

すべてのサービスが無料枠内で運用されています。

### サービス別コスト分析

| サービス | 無料枠 | 現在の使用量 | 超過時コスト | リスク |
|----------|--------|-------------|-------------|--------|
| Firebase Auth | 50,000回/月 | 数百回 | $0.02/50K回 | 低 |
| Firestore 読み込み | 50,000回/日 | 数百回 | $0.18/100K回 | 中 |
| Firestore 書き込み | 20,000回/日 | 数百回 | $0.06/100K回 | 中 |
| Firestore ストレージ | 1GB | 数MB | $0.18/GB/月 | 低 |
| GitHub Gist API | 60回/時 | 1-3回/セッション | 無料（制限のみ） | 低 |
| Google Apps Script | 1000万回/月 | 数十回 | 無料 | 低 |
| GitHub Pages | 1GB帯域/月 | ~100MB | 無料 | 低 |
| GitHub Actions | 2,000分/月 | ~200-300分 | 無料 | 低 |

### スケール時のコスト予測

**唯一のコスト変動要因は Firestore です。**

| DAU | 書き込み/日（推定） | 月額コスト |
|-----|-------------------|-----------|
| 100人 | ~1,000回 | **0円** |
| 1,000人 | ~10,000回 | **0円** |
| 3,000人 | ~20,000回 | **0円（上限付近）** |
| 5,000人 | ~35,000回 | **~$900/月** |
| 10,000人 | ~70,000回 | **~$3,000/月** |

※ 1ユーザーあたり平均5-7回/日の書き込みを想定

### コスト発生閾値

```
DAU 3,000人 ≒ Firestore 無料枠の上限
```

この規模では月額サブスク収益（200円 × 契約率）で十分カバー可能です。

## データフロー

### 認証フロー
```
初回アクセス → 匿名認証（自動）→ UID 発行
    ↓
メール連携（任意）→ 匿名アカウントにメール/パスワードを紐づけ
    ↓
別端末でログイン → 同じ UID でデータ復元
```

### データ同期フロー
```
ユーザー操作（データ変更）
    ↓
ローカル DB（SQLite WASM）に即時書き込み
    ↓
SyncManager.requestSync()（3秒デバウンス）
    ↓
SHA-256 ハッシュ比較（前回と同じならスキップ）
    ↓
全データ JSON エクスポート → Firestore にアップロード
```

### サブスクリプション検証フロー
```
アプリ起動時
    ↓
verifySubscription()（Apps Script 経由で Stripe に問い合わせ）
    ↓
Stripe の実契約状態を取得 → ローカル状態を同期
    ↓
プレミアム画面アクセス時（5分間隔スロットル）
    ↓
バックグラウンドで再検証
```

## セキュリティ対策

| 対策 | 内容 |
|------|------|
| サブスク検証 | サーバー検証必須化（URL パラメータだけでは有効化不可） |
| 空データ上書き防止 | ローカル DB が空の場合、クラウドバックアップを上書きしない |
| DB マイグレーション | スキーマ変更時は必ず `schemaVersion` + `onUpgrade` をセットで更新 |
| 入力バリデーション | XSS サニタイズ、パラメータ化クエリ |
| CORS | Google Apps Script がプロキシとして CORS を処理 |
